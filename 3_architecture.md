# Архитектура Email-аутрича на 1200 адресов

## Задача
Обслуживать 1200 email-адресов для аутрича (несколько клиентов, несколько направлений) с минимальной стоимостью и высокой отказоустойчивостью.

## Архитектура

### Инфраструктура email
- **12 доменов** (по 100 ящиков на домен — безопасный лимит)
- **Хостинг**: Namecheap Private Email ($1.5/ящик/мес) или Zoho Mail ($1/ящик/мес)
- **Альтернатива**: Google Workspace ($6/ящик) для критичных клиентов

### Сравнение провайдеров

| Провайдер | Лимит/день | Цена/ящик | Подходит для bulk? |
|-----------|------------|-----------|-------------------|
| Zoho Mail | 200-1000 | $1 | Условно (риск бана) |
| Namecheap | ~500 | $1.5 | Да, простой |
| Google WS | 2000 | $6 | Высокая deliverability |

> ⚠️ **Важно**: Zoho запрещает cold emails в ToS — риск suspension. Использовать с VPN/ротацией IP.

### Прогрев и репутация
- **Warmup сервис**: Instantly.ai или Smartlead ($97-197/мес за 1000+ ящиков)
- **Период прогрева**: 14-21 день, старт с **5-10 писем/день**, рост до 50/день
- **SPF, DKIM, DMARC** — обязательно на всех доменах
- **Тренд 2026**: AI-оптимизация времени отправки (Smartlead), multi-channel (email + LinkedIn)

### Отправка
- **Основной инструмент**: Instantly.ai / Smartlead / Lemlist
- **Лимиты**: 30-50 писем/день/ящик = 36,000-60,000 писем/день максимум
- **Smartlead** дешевле Lemlist для unlimited inboxes

### Ротация и распределение
```
Клиент A (направление 1) → Пул 1 (200 ящиков, 4 домена)
Клиент A (направление 2) → Пул 2 (200 ящиков, 4 домена)
Клиент B → Пул 3 (400 ящиков, 4 домена)
Резерв → 400 ящиков (30% на замену выгоревших)
```

### Мониторинг
- **Bounce rate**: алерт при >5%
- **Spam complaints**: алерт при >0.1%
- **Deliverability**: проверка через GlockApps ($59/мес)
- **Автоматическая ротация**: выгоревший ящик → в карантин → замена из резерва

При Google Workspace: ~$8,500/мес (дороже, но выше deliverability)

## Риски и митигация

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Массовый бан домена | Средняя | Изоляция клиентов по доменам, резерв 30% |
| Попадание в спам | Высокая | Прогрев, мониторинг, A/B тесты subject |
| Блокировка IP | Низкая | Используем SaaS, не свои сервера |
| Жалобы получателей | Средняя | Unsubscribe в каждом письме, чистка базы |
| Zoho suspension | Средняя | VPN/ротация IP, не превышать лимиты |

## Масштабирование

- До 3000 ящиков: добавляем домены и ящики линейно
- 3000+: переход на выделенные IP + собственный SMTP (Mailgun/Postmark)
- 10000+: распределённая инфраструктура, несколько провайдеров

## Вывод

Для 1200 ящиков оптимально использовать SaaS-решения (Instantly/Smartlead + дешёвый email-хостинг). Это даёт:
- Быстрый старт (1-2 дня на настройку)
- Встроенный warmup и ротацию
- Минимальные DevOps затраты
- Стоимость ~$1.2/ящик/мес

Собственная инфраструктура имеет смысл только при 5000+ ящиков или специфических требованиях к контролю.
